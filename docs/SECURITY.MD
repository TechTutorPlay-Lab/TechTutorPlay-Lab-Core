# Security Policy 🛡️

## Active Monitoring
This repository is protected by **GitHub Advanced Security (GHAS)**.

* **Secret Scanning:** Automatic blocking of pushes containing API keys, tokens, or credentials.
* **CodeQL Analysis:** Periodic scanning for logical vulnerabilities and security flaws.
* **Dependency Review:** Monitoring for vulnerabilities in third-party libraries.

## Best Practices
1. **Never commit secrets:** Use environment variables or secret managers.
2. **Local Security:** Development occurs in protected virtual environments.
3. **Audit Logs:** Regular review of access and security reports provided by the Enterprise dashboard.
